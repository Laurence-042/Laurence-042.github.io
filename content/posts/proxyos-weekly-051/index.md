## 内部编辑器的慢速执行模式

实际上这本质上几乎是我之前认为不合理的路线，也就是“自造类 vs code 编辑器”，当时我的核心观点是“玩家不会希望自己使用一个不方便的编辑器来写代码”

但我当时忽略了一个事实，那就是 vs code 虽然与 IDE 相比十分简单、模块化，但是对只用过office甚至连office都没用过的初学者来说，依然是一套新的概念

初学者恐怕很难理解左侧边栏那一大堆按钮，也很难接受“F5调试”的习惯

因此最佳实现应该时同时提供两者：在教程阶段提供更适合教学的、功能较少但提示较多的定制内部编辑器。当玩家之后也可以继续使用内部编辑器，但是如果玩家觉得内部编辑器开始限制他们了，他们也可以无摩擦地使用vs code等外部编辑器

所以本期为内部编辑器添加了诸如慢速执行等QoL功能，方便玩家更清楚地看出执行过程

## 关于玩家Alt+F4的可能
本期对序章的引入做了大量可靠性加固，但是感觉当前的剧本系统有些力不从心了。

这主要是因为当前系统是 以任务完成/触发为主的触发器+各种后效列表 构成的，如果后效执行过程中被停止，那么还未执行的后效都会丢失。

更可靠的系统可能是类似有向图，每次完全执行后才进行存档，这样就能保证操作的原子性。

但具体怎么实现我还得再想想。
